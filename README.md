# AMA-teras Community Registry(進化レジストリ)

[AMA-teras](https://github.com/moriwo-dev-ai/ama-teras) 本体の「自己進化」で生まれたツールプラグインを、
コミュニティで共有するためのレジストリです。コンセプトは **「作る前に探す」** —
本体アプリはツールを新規生成する前にこの索引を検索し、既存のプラグインがあれば
生成コストをかけずに導入を提案します。

## 使い方(導入する側)

**このリポジトリを直接 clone する必要はありません。** AMA-teras 本体の
設定 → 接続タブ → 「コミュニティレジストリURL」に本リポジトリの raw ベースURL
(例: `https://raw.githubusercontent.com/moriwo-dev-ai/amateras-registry/main`)を設定するだけで:

1. エージェントが新ツールを作ろうとしたとき、既存プラグインを自動で提案します
2. 承諾すると本体がダウンロードし、**ローカルの検証ゲート(B環境で typecheck → テスト →
   スモーク)と昇格承認ダイアログ**を通ってから導入されます — レジストリ掲載の有無に
   かかわらず、無審査でコードが動くことはありません

## 神(運営エージェント)も配布しています

AMA-teras の「運営」機能で働く**神エージェント**は、コードではなく**定義データ**です
(`{id, name, engine, clock, judgePrompt?, dailyTokenBudget, enabled}`)。
そのためレジストリではJSON 1枚で配布でき、本体アプリの
**運営タブ → AMENO-koyane → 「⛩ レジストリから神を迎える」** から検索して迎えられます。

迎え入れは必ず**岩戸ゲート**(定義JSONの全文を見せる承認ダイアログ)を通ります。
承認しなければ1体も増えません。配布される定義は必ず `"enabled": false`(停止状態)で始まり、
動かすかどうかは迎えた人が決めます。

`engine` は本体が実装している5種のみです(`metrics-observer` / `community-patrol` /
`draft-writer` / `issue-gatekeeper` / `kamuhakari`)。**新しいエンジンは本体のコード変更**であり、
レジストリでは配れません(本体リポジトリへ Issue / PR をどうぞ)。

## 構造

```
index.json            … プラグインと神の索引(本体アプリが検索に使う)
revoked.json          … 失効リスト(キルスイッチ。危険が判明したプラグインを無効化)
plugins/<name>/       … プラグイン1つ = 1ディレクトリ
  <name>.ts           … プラグイン本体(ToolPlugin 規約)
  <name>.test.ts      … vitest ユニットテスト(必須)
  manifest.json       … マニフェスト(権限宣言・ライセンス・互換範囲)
gods/<id>/            … 神1体 = 1ディレクトリ
  <id>.json           … 神の定義(コードではなくデータ。秘密は含めない)
scripts/validate.mjs  … CI検証スクリプト(マニフェスト検証+権限の静的解析+神定義の検証)
```

## 掲載の2層

| 層 | 意味 | 導入時の扱い |
|---|---|---|
| **検証済み**(`"verified": true`) | CI全ゲート+人間のレビュー承認を通過 | 候補として通常表示 |
| **未検証**(`"verified": false`) | 掲載のみ(CIは通過) | 追加の警告付きで表示 |

## 投稿手順(共有する側)

1. このリポジトリを**フォーク**する
2. AMA-teras 本体のツール一覧 → 「📦 エクスポート」で書き出したディレクトリを
   `plugins/<name>/` に配置する(コード+テスト+manifest.json の3点セット)
3. `index.json` にエントリを追記する(`verified` は `false` のまま。検証済みへの昇格は
   メンテナーのレビュー後)
4. **PRを作成**する(PRテンプレートのチェックリストと DCO 同意に従ってください)

CI がマニフェスト検証と権限の静的解析(宣言外API使用の検出)を自動実行し、
結果をPRに表示します。詳細は [CONTRIBUTING.md](CONTRIBUTING.md) を参照してください。

### 神を投稿する場合

1. 本体の 運営タブ → 「神の定義」→ **「📤 定義を書き出す」** で `<id>.json` を書き出す
2. `gods/<id>/<id>.json` に置き、`"enabled": false` にする(迎えた人が自分で有効化する)
3. `index.json` の `gods[]` にエントリを追記して**PR**(命名は日本神話=八百万の神から)
